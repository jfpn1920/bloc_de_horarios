# 👋👋 ¡Bienvenidos! a mi proyecto bloc de horarios

<img src="imagen_presentacion.png" alt="Presentación" width="220" align="left" style="margin-right: 20px;">

<p style="text-align: justify;">
¡Bienvenido a este repositorio! Aquí encontrarás la presentación de mi proyecto <b>bloc de horarios</b>, una propuesta enfocada en la organización, planificación y administración de actividades mediante diferentes tecnologías y enfoques de programación. Este proyecto fue desarrollado como ejercicio de aprendizaje utilizando <b>diversos programas y lenguajes de programación</b>, permitiendo fortalecer conocimientos en lógica de programación, estructuras de datos, desarrollo web, programación cliente-servidor y bases de datos.
</p>

<p style="text-align: justify;">
    A lo largo del proyecto se implementaron diferentes versiones del mismo sistema, comenzando desde aplicaciones de consola hasta una aplicación web completa con persistencia de datos utilizando <b>localstorage</b> y posteriormente almacenamiento permanente mediante <b>xampp y phpmyadmin</b>. De esta manera, el proyecto permite comprender la evolución progresiva de un sistema de gestión de horarios desde algoritmos básicos hasta una solución full stack funcional.
</p>

### ✨ Características del proyecto
* Gestión organizada de horarios y actividades.
* Desarrollo progresivo del mismo sistema en múltiples lenguajes de programación.
* Aplicaciones de consola e interfaz web funcional.
* Almacenamiento local utilizando <b>localstorage</b>.
* Integración completa entre <b>frontend y backend</b>.
* Implementación de base de datos con <b>xampp y phpmyadmin</b>.
* Operaciones crud funcionales (crear, leer, actualizar y eliminar horarios).
* Actualización dinámica de registros en tiempo real.
* Seguimiento de actividades programadas.
* Visualización detallada de horarios registrados.
* Vista previa de cada horario.
* Edición y eliminación dinámica de registros.
* Tablero interactivo de horarios.
* Gestión de estados de actividades.
* Organización de tareas por fecha y hora.
* Manipulación dinámica del dom con javascript.
* Comunicación entre frontend y backend mediante <b>fetch api</b>.
* Manejo de datos en formato <b>json</b>.
* Persistencia permanente mediante <b>mysql</b>.
* Proyecto desarrollado sin frameworks.
* Organización estructurada de archivos y código.
* Enfoque orientado al aprendizaje progresivo de programación.
